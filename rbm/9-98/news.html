Cross-Platform Compiling

  In DR2r5, a new feature was added to the Build Application dialog: x86. This checkbox, which appears under the 68K checkbox, allows you to build applications for Windows. Though it isn't very effective (it crashes on anything that isn't extremely simple), it does work. However, you can only create 32-bit code, which means that applications compiled for windows always require Windows 95, 98, or NT.

Navigation Services in RB

  One new feature in Mac OS 8.5 (Allegro) is the revamped Open & Save boxes. Though you can't access these through commands built-in to RB, you can access them through an XCFN called "Full Navigation". This XCFN allows you to access all of the six (you read right, six) types of dialogs provided by the Navigation services.
  This may not be useful for long though; I've been informed by Real SW that support for Nav Services will be built-in by the time OS 8.5 is released in October.

Anti-Aliased Text for RB

  There's a new module available for download at the RB bbq and café which anti-aliases text in any font, size and style onto any background. How? If you've ever tried to make a smooth routine for graphis in RB, you'll realize quickly that you can't possibly make it fast enough. So how does this module do it? Actually, the process of anti-aliasing has nothing to do with going through all the pixels of an image. You draw the text at four times its actual size, and then shrink it down. The resulting image uses shades of gray to make text much smoother.

Mozilla, here we come!

  James Milne (who I enjoy picking on) will soon be coming out with an HTML decoder plug called "MozPlug".
  This plugin, which is based on the recently released Netscape 4.0 source code, will offer support of all the HTML tags that Netscape 4.0 supports, including JavaScript and Frames.
  If it's used in conjuction with httpSocket, this could make for some very powerful programs.

ActionSurface, now we can see thee!

  My ActionSurface class, which used to be protected, is no longer so. This change was made because of the nature of the class. In a way, it's designed to be changed. The form it comes in should only be a basis for any games you make.
  In case you don't know what ActionSurface is, it's a subclass of the SpriteSurface control that is geared toward side-scrolling action games like Mario, where you are a main character, and you jump around, run into walls, etc. Essentially, you define platforms, walls, enemies, and the main character, and it takes care of the rest. It's available at the RB café and BBQ.

OS 8 List boxes are here.

  A new feature in DR2r8 is the "HasHeader" property for Listboxes. If this is checked, then the listbox will have a seperate, clickable header for each column. When clicked, the contents of the list box are sorted by that column. It does leave much to be desired however. It's not the same as the ones in Mac OS 8 list boxes, and instead of the "reverse sort" button, there's an empty square. Also, you can't sort numerically.
